# SUS 与 CHUNITHM c2s 格式整理及转换思路（草案）

本文档整理 **SUS（Sliding Universal Score）** 与 **CHUNITHM 谱面 `.c2s`** 的结构差异，并给出从 SUS 走向 c2s 的**分阶段实现思路**。不涉及具体代码实现。

## 2. SUS 格式概要（基于 v2.7 规格）

SUS 是 **UTF-8 文本**，以 `#` 开头的行为结构化数据，其余行多视为注释。

### 2.1 元数据行

常见指令包括（非穷尽）：

- `#TITLE` / `#ARTIST` / `#DESIGNER` / `#DIFFICULTY` / `#PLAYLEVEL` / `#SONGID`
- `#WAVE` / `#WAVEOFFSET` / `#JACKET` / `#BACKGROUND` / `#MOVIE` 等资源路径
- `#BASEBPM`、`#BPMzz` 与 `#mmm08` 形式的 BPM 定义与引用
- `#REQUEST`：如 `ticks_per_beat`（每拍 tick 数，影响小节内分割）、`enable_priority` 等

PJSK 导出的 SUS 常带 `#REQUEST "ticks_per_beat 480"` 等，与街机/编辑器默认 tick 可能不同，**转换时必须先统一时间基**。

### 2.2 谱面数据行（`#mmmxy` / `#mmmxyz`…）

- 行结构：**头部 + `:` + 数据部**。
- **数据部**按 **每 2 个字符为一组** 将小节划分为多个时间点；组内第 1 位表示音符种类/状态，第 2 位表示**宽度**（1～35 常用 `1`～`z` 编码）。
- **小节号** `mmm` 从 0 起算；`x` 多表示**左端轨/列**（0–9、a–z）；`y` 为**通道**（同套字母数字），用于 Hold/Slide 等同通道相连。

规格中与本转换最相关的通道类型包括（与 kb10uy v2.7 一致；**键型 → c2s 事件映射**见同目录 `sus_to_c2s_note_mapping_zh.md`）：


| 通道模式      | 含义（简化）                      |
| --------- | --------------------------- |
| `#mmm1x`  | Tap（`1`–`6` 等子类型对应不同 tap）   |
| `#mmm2xy` | Hold：起点 / 中继 / 终点（同通道相连）    |
| `#mmm3xy` | Slide 1（可移动长条，含贝塞尔控制点等）       |
| `#mmm4xy` | Slide 2（与 Slide 1 同类拓扑，实现差异依数据源） |
| `#mmm5x`  | Directional（方向键；上/下/四斜向）    |
| `#mmm02`  | 小节拍长（可小数，宜满足 M/2^n 形式以利于分割） |


另有 `#TIL` / `#HISPEED`、`#ATR` / `#ATTRIBUTE`、`#MEASUREBS` / `#MEASUREHS` 等，影响显示与局部速度；**完整转换需逐条语义对齐或明确「不支持子集」**。

### 2.3 与 PJSK 谱面的关系

游戏内扒出的 PJSK SUS 常经工具（如 [PjskSUSPatcher](https://github.com/Qrael/PjskSUSPatcher)）去掉技能线等；其 **Tap/Hold/Slide/Directional** 仍符合 SUS 通道思想，但 **轨数、技能、fever 等与 CHUNITHM 不同**。转换属于**跨游戏重映射**，不是单纯语法替换。

## 3. c2s 格式概要（结合示意图与 `2896_04.c2s`）

c2s 在本样本中为 **制表符或空白分隔的文本行**（非二进制整体封装的一层事件表），首部为**全局定义**，其后为**按时间排列的事件**。

### 3.1 文件头（`2896_04.c2s` 实例）

示例字段：

- `VERSION`：如 `1.13.00`
- `MUSIC` / `SEQUENCEID` / `DIFFICULT` / `LEVEL` / `CREATOR`
- `BPM_DEF`、`MET_DEF`（拍号）
- `RESOLUTION`、`CLK_DEF`：时间与 tick 的基准（本文件为 **384**）
- `BPM`、`MET`：具体小节上的 BPM/拍号事件
- 其它如 `PROGJUDGE_*`、`TUTORIAL` 等

**结论**：写 c2s 前必须固定 **RESOLUTION/CLK** 与 **BPM/MET 时间轴**，再把每个音符映射到 **（小节, tick）**。

### 3.2 事件行：类型与参数（示意图 + 样本对照）

示意图将多种类型归纳为「小节 + tick + 位置 + 长度」为公共前缀，其后按类型附加 **持续时间、结束位置/长度、终点音符类型、常量 5.0** 等。真实文件中字段为 **TAB 分隔的列**，列数随类型变化，且部分行末尾带 `**SLD` / `DEF` / `TAP` / `ASC` / `L` / `UP`** 等**标签或子类型枚举**，与示意图中的「最后一列常数 1」并不总是一致——**以真实 c2s 与引擎行为为准，示意图作理解辅助**。

结合示意图与 `2896_04.c2s` 的对应关系（概念级）：


| 代码                                            | 示意图中的角色       | 样本中可见特征（简述）                                                        |
| --------------------------------------------- | ------------- | ------------------------------------------------------------------ |
| `TAP`                                         | 普通红键          | 多列整数，地面单点                                                          |
| `FLK`                                         | 蓝键 / Flick    | 常带方向后缀如 `L`                                                        |
| `CHR`                                         | 黄键 / 侧键类      | 可带 `UP` 等                                                          |
| `HLD`                                         | 长按            | 较 `TAP` 多 **持续时间**（tick）                                           |
| `SLC` / `SLD`                                 | 滑条段           | `SLC` 与 `SLD` 成链出现，含多组位置/宽度/时间，末列可出现 `SLD` 引用下一段                   |
| `SXC` / `SXD`                                 | 另一套滑条变体       | 示意图列出，本样本以 SLC/SLD 为主，需另谱面对照                                       |
| `AIR` / `ADW` / `ADR` / `ADL` / `AUL` / `AUR` | 天空方向键         | 示意图；样本见 `ADL` 与 `FLK` 等连用                                          |
| `ALD`                                         | 大天空条          | 示意图含持续时间                                                           |
| `ASD` / `ASC` / `AHD`                         | 空中线/斜线/空 hold | 样本中 `ASC`/`ASD` 行较长，含 `**5.0`**、`DEF`、嵌套指向 `TAP`/`ASC` 等，**语义最复杂** |


**要点**：c2s 把一条「长滑条」拆成 **多行 `SLC` + `SLD`** 的链式结构；SUS 的 `#mmm3xy` 则用 **同通道多控制点** 描述一条几何曲线。**二者拓扑不同，不能 1:1 按行翻译。**

### 3.3 谱面流动速度：`SLP` 与头部时间基准（实例 `ACUS/music/music7009/7009_04.c2s`）

该谱为 **VERSION 1.13.00**，`RESOLUTION` / `CLK_DEF` 均为 **384**，开头有 `BPM_DEF` / `MET_DEF` 与一行 **`BPM` / `MET`** 定初始 **220 BPM、4/4**。与 `2896_04` 不同之处在于：全谱 **仅这一处 `BPM`**，**中盘不改 BPM**；大量 **变速**写在 **`SLP`** 行，控制的是 **谱面滚动（Hi-Speed / 流速倍率）** 随时间的区段，而非音符事件本身类型。

#### 3.3.1 `SLP` 行格式（按本文件归纳）

每行：**`SLP`** + **5 列**（TAB 分隔）：

| 列 | 名称（归纳） | 含义 |
|----|----------------|------|
| 1 | `measure` | 小节号，整数，从 0 起 |
| 2 | `tick` | 该小节内时间，与 `RESOLUTION` 同一套 tick |
| 3 | `length` | **持续长度（tick）**：该条规则作用的时间跨度 |
| 4 | `speed` | **速度倍率**（浮点）；见下文物殊值 |
| 5 | `timeline` | **Hi-Speed / 流速时间线编号**（整数）。与 MRGC 侧按 `#TIL` 等分组的 **ScrollSpeed** 轨道一致；单轨谱常见为 `0`。[PenguinTools](https://github.com/Foahh/PenguinTools) 的 `Slp.Timeline` 即此列（`C2SConverter.Event.cs` 中按 `Timeline` 分组后再生成相邻区段 `SLP`）。 |

**阅读方式**：在「`(measure, tick)` 起、持续 `length` tick」的区间内，谱面流动使用约 **`speed` 倍**于某默认流速（与引擎内部基准相关；实现时以对照官谱手感为准）。

#### 3.3.2 特殊值 `10000.0`

本谱多次出现 **`10000.000000`**，且往往搭配 **很短的 `length`（0、1、96 等）** 或与正常倍率（如 `0.9`）**紧挨**出现。

- **不应**按字面理解为「一万倍速」。
- **合理推测**（需游戏内或逆向确认）：表示 **区段边界 / 重置 / 立即切换** 等与引擎状态机相关的**哨兵值**，或「关闭某种插值、改用下一段定义」一类语义。

SUS→c2s **若无法从 `#TIL`/`#HISPEED` 推导**：要么 **不生成 `SLP`**（与官谱手感差异大），要么 **保留目标难度模板中的 `SLP` 骨架** 再填音符。

#### 3.3.3 本谱中可见的几种变速模式

1. **长区段恒定倍率**  
   - 例：`SLP 0 0 1920 0.600000 0`：从第 0 小节第 0 tick 起，**连续 1920 tick** 使用 **0.6** 倍速。  
   - 同类：`6 0 768 0.6`、`8 0 384 0.8` 等。

2. **短区段 + 哨兵 + 下一段正常倍率**  
   - 例：`9 0 96 10000` 后接 `9 96 288 0.9`：前 96 tick 为特殊段，随后 288 tick 为 **0.9** 倍速。用于**切段**或**与上一区段脱钩**。

3. **同一小节内「楼梯」离散降速**  
   - 例：小节 **55**：在 `55 0 0 10000` 之后，从 `55 0 12 5.0` 起每隔 **12 tick** 写一行，倍率从 **5.0** 逐级降到 **1.25**（5.0 → 4.75 → … → 1.25）。  
   - 效果：**用多段常数逼近连续减速**（台阶越细越像平滑变速）。

4. **极低倍率（近似停谱）**  
   - 例：`81 132 60 0.010000 0`：在一段 **60 tick** 内使用 **0.01** 倍速，主观上接近 **几乎不动**。

5. **跨小节大块阶梯 + 小节内细阶梯**  
   - 例：小节 **98～100**：半小节或整小节切换 **0.9/0.8**、**0.7/0.6**、**0.5** 等。  
   - 例：小节 **101**：每 **48 tick** 升一档，从 **0.55** 逐步升到 **0.95**（**加速楼梯**）。

6. **尾部极短哨兵**  
   - 例：`109 160 1 10000`：`length` 仅 **1** tick，可能表示**段落结束或衔接**。

#### 3.3.4 与 SUS 的对应关系（供转换文档化）

| SUS / 通用 | c2s 中与本节相关 |
|------------|------------------|
| `#BPMzz`、`#mmm08` | `BPM`（可多行；本实例仅一行） |
| 小节长度 `#mmm02`、拍号 | `MET`、以及 tick 换算 |
| `#TIL`、`#HISPEED`、`#MEASUREHS` | 语义上最接近 **`SLP` 区段表**；**字段与引擎语义均不对齐**，需单独设计映射或手工对齐官谱 |

更细的 **键型→事件** 仍以 `docs/sus_to_c2s_note_mapping_zh.md` 为准；**流速**建议作为 c2s 的**独立第二轴**实现，与音符解析器解耦。

### 3.4 BPM 变化与拍号变化（`BPM` / `MET` / `*_DEF`）：对照 PenguinTools

以下与 [Foahh / PenguinTools](https://github.com/Foahh/PenguinTools) 中 **`PenguinTools.Core/Chart`** 一致：`C2SConverter.cs` 写头部与事件表，`Models/c2s/Events.cs` 定义各事件**写出时的列顺序**，`C2SConverter.Event.cs` 把 MRGC（`mgxc`）里的 BPM / 拍号 / 流速事件展开为 c2s 行；`TimeCalculator.cs`（改编自 [Ched](https://github.com/paralleltree/Ched)）说明 **拍号变化后小节长度（tick）如何递推**。

#### 3.4.1 头部 `BPM_DEF` / `MET_DEF`

- **`BPM_DEF`**：工具将 **主 BPM** 以 **同一数值写满 4 列**（`MainBpm:F3` 重复四次），与样本 `2896_04.c2s` 等形式一致。
- **`MET_DEF`**：**两列**依次为 **`denominator`**、**`numerator`**（与行内 `MET` 后两列顺序一致），表示初始拍号。

其后常接 **`RESOLUTION`** / **`CLK_DEF`**（该工具当前生成 **384**；与本文 §3.1 样本一致）。时间换算在工具内部另用 **MAR 分辨率 1920** 与 c2s **384** 的比例关系做舍入（见同目录 `Models/Time.cs` 的 `Round` / `Scaled` / `Position`）。

#### 3.4.2 行内 `BPM`（可多行）

- **列结构**：`BPM`、**小节号**、**小节内 offset（tick，与 `CLK_DEF` 一致）**、**BPM 值**（三位小数）。对应 `Bpm` 类：`Text` 为 `Id` + `Position` + `\t` + `Value:F3`。
- **来源**：每个 `mgxc.BpmEvent` 输出一行，按 tick **排序**写入事件区。
- **语义**：谱面可在不同 `(measure, offset)` 多次声明 BPM，实现 **变速**；与 §3.3 的 **`SLP` 谱面流速** 是不同维度——例如 `7009_04` 仅一处 `BPM` 而大量 `SLP`，即 **音乐 BPM 不变、仅滚动速度变**。

#### 3.4.3 行内 `MET`（拍号变化，可多行）

- **列结构**：`MET`、小节号、小节内 offset、**分母**、**分子**。  
  **注意**：`Met.Text` 明确为 **`Denominator` 在前、`Numerator` 在后**（与乐理简写「4/4」先写分子再斜杠的习惯不同，但与 `MET_DEF` 两列顺序相同）。实现或手改 c2s 时勿把两列对调。
- **来源**：每个 `mgxc.BeatEvent`（拍号事件）一行，按 tick 排序。
- **小节长度（tick）**：在固定 `RESOLUTION`（如 384）下，**当前拍号**下 **一小节的 tick 数**可归纳为：

  \[
  \text{measureLength} = \text{RESOLUTION} \times \frac{\text{numerator}}{\text{denominator}}
  \]

  拍号在中间变更时，**变更点之后**应用新拍号计算小节边界；`TimeCalculator` 用「自上一拍号事件起的 tick 差 ÷ 上一段每小节长度」累加 **小节序号**，用于把全局 tick 映射到 `(bar, beat, tickOffset)`。

#### 3.4.4 与 SUS 的对应（文档层）

| SUS（常见） | c2s |
|-------------|-----|
| `#BASEBPM`、`#BPMzz`、`#mmm08` 等 BPM 定义链 | `BPM_DEF` + 多条 `BPM`（变化时刻对齐到统一 tick 轴后写出） |
| `#mmm02` 小节拍长、拍号相关元数据 | `MET_DEF` + 多条 `MET`；**每段拍号**下重算小节 tick 总长与事件落点 |

#### 3.4.5 流速类事件的并列写法（工具行为摘要）

同一套 `C2SConverter.Event.cs` 在写出 `BPM`/`MET` 之外，还会：

- 由 **`NoteSpeedEvent`** 一类事件生成 **`DCM`** 区段（与音符侧速度相关；模型见 `Events.cs` 中 `Dcm`）。
- 由 **`ScrollSpeedEvent`**（带 `Timeline`）生成 **`SLP`**，且 **丢弃 `speed == 1` 的区段** 后再写入。

SUS→c2s **自研映射**时，可只实现 **`BPM`/`MET` + 音符**，暂不生成 `SLP`/`DCM`；但若要与官谱手感一致，需另建 §3.3 所述 **流速轴**。

## 4. 时间与空间：对齐思路

### 4.1 时间轴

- **SUS**：小节号 + 数据部内 **2 字符一组** 的细分 + `ticks_per_beat` / 小节拍长 `#mmm02`。
- **c2s**：**小节整数 + tick 整数**，tick 最大值与小节长度由 `RESOLUTION`、拍号、BPM 共同约束。

**转换步骤（概念）**：

1. 解析 SUS 的 `ticks_per_beat` 与每小节拍数，建立 **「全局 tick」或「小节+子刻度」** 统一坐标。
2. 选定目标 c2s 的 `RESOLUTION`（如 384），将 SUS 上每个事件中心时刻 **量化/取整** 到该分辨率。
3. 输出 `BPM`/`MET` 行，使与 SUS 中 BPM、拍号变化点一致（`MET` 列顺序见 §3.4.3）；或接受简化：单 BPM + 固定 4/4。

### 4.2 空间（轨 / 宽）

- **SUS**：左端轨 `x` + 宽度字符（第二位）。
- **c2s**：`位置` + `长度`（示意图用语）在样本中对应 **整数列**。

**已定方案**（详见 `docs/sus_to_c2s_note_mapping_zh.md` §1.5）：PJSK **12 轨**（左端索引 0～11）映射到中二 **16 轨中的中间 12 轨**，省略左右各 2 轨：`chuni_left = pjsk_0based + 2`（左端落在列 2～13）；宽度越界时钳位或告警。

## 5. 初步转换思路（分阶段）

建议按难度递进，便于验证。

### 阶段 A：元数据与空壳 c2s

- 从 SUS 读取 `TITLE`/`ARTIST`/`PLAYLEVEL` 等写入 c2s 头（字段名不完全一致处做表驱动映射）。
- 固定 `VERSION`、`RESOLUTION`、`CLK_DEF`、`BPM_DEF`/`MET_DEF`，生成 **无音符** 的可加载 c2s，验证工具链。

### 阶段 B：仅 Tap（`#mmm1x` → `TAP`）

- 忽略 Hold/Slide/Directional。
- 完成时间量化与轨映射后，对每条 tap 输出一行 `TAP`。
- 用 CHUNITHM 编辑器或游戏内检视对齐节奏。

### 阶段 C：Hold（`#mmm2xy` → `HLD`）

- 解析同通道 `1` 起点、`3` 中继、`2` 终点，计算 **持续 tick**。
- 注意 SUS 宽度沿条恒定；与 c2s `HLD` 各列对齐。

### 阶段 D：Slide（`#mmm3xy` / `#mmm4xy` → `SLC`/`SLD` 链等）

- 将 SUS 一条 slide 拆成 c2s 多段：需参考 **官方 c2s**（如本仓库 A001 多谱）或 **PenguinTools** 中由 MRGC 生成 c2s 的拆条规则。
- 难点：**贝塞尔控制点**与 **SLC 折线/样条** 的近似误差。

### 阶段 E：Directional / Air / 复合（`#mmm5x` 与示意图中的 `AIR`/`AD`*/`ASC`/`ASD`/`AHD`）

- PJSK 与 CHUNITHM 天空层规则差异大；建议在阶段 B–D 稳定后再做。
- `ASC`/`ASD` 行长、末尾 `DEF`/`5.0` 的含义建议 **对照 PenguinTools 源码或更多 c2s 样本** 做逆向表，而非仅凭注释图。

## 6. 与 PenguinTools 的协作方式（建议）

## 7. 风险与限制（简要）

- **跨游戏**：PJSK SUS 并非为 CHUNITHM 编写，无理键型、密度、天空规则需人工或规则库兜底。
- **规格版本**：c2s `VERSION` 与字段随游戏版本演变；应以目标版本（如 A001 样本的 1.13.x）为 **golden file** 做回归。
- **示意图与实现**：注释图可能对应特定解析器版本；**最终以官方/样本 c2s 列顺序与枚举为准**。
