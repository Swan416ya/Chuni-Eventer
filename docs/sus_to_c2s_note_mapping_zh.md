## 1. SUS 数据行通道号（kb10uy v2.7 摘要）

一行形如 `#<小节><通道类型><轨x>[通道y]: <数据部>`。**通道类型**决定该行语义（与数据部首字符组合使用）：

| 头部模式 | 规格含义 | 数据部首字符角色（摘要） |
|----------|-----------|---------------------------|
| `#mmm1x` | **Tap**（单点） | `1`–`6`：不同 Tap 子类型；次字为宽度 |
| `#mmm2xy` | **Hold** | `1` 起点 / `2` 终点 / `3` 中继 |
| `#mmm3xy` | **Slide 1**（可移动长条，含控制点） | `1` 起 / `2` 终 / `3` 可见中继 / `4` 贝塞尔控制 / `5` 不可见中继 |
| `#mmm4xy` | **Slide 2** | 与 Slide 1 同类拓扑，具体差异依实现；**与 CHUNITHM 哪类滑条对应待你定** |
| `#mmm5x` | **Directional**（方向键） | `1` 上 / `2` 下 / `3` 左上 / `4` 右上 / `5` 左下 / `6` 右下 |
| `#mmm02` | 小节拍长 | 非音符 |

> **注意**：部分 PJSK 工具或旧谱可能把「方向类」写在别的通道上；若你发现与上表不符，请在 §0 注明数据源，并在 §4 增加例外行。

## 1.5 轨位映射：PJSK 12 轨 → 中二中间 12 轨（已定）

- **中二地面**按 **16 个位置列**理解（与 c2s 里「位置 + 长度」的离散格一致；具体数值以 `RESOLUTION` 与样本为准）。  
- **不使用**最左 **2** 列与最右 **2** 列，只使用 **中间 12 列**，与 **PJSK 的 12 轨** 一一对应。  
- **0-based 列索引**：中二整宽为 `0…15`；实际写入 c2s 的左端列取 **`2…13`**（共 12 列）。

令从 SUS 解析得到的 PJSK **左端轨索引**为 `p`（**0～11**，对应 PJSK 第 1～12 轨；若你的解析器用 1～12 请先化为 0～11）：

```text
chuni_left = p + 2
```

| PJSK 轨 `p`（0-based） | 中二左端列（0-based） |
|------------------------|-------------------------|
| 0 | 2 |
| 1 | 3 |
| … | … |
| 11 | 13 |

**宽度**：SUS 数据部每组第 2 个字符表示宽度（1～35 的编码）。将宽度换为 **格数** `w` 后，应满足 `chuni_left + w ≤ 14`（即不侵入最右未使用两列 `14、15`）；若越界则 **钳位** 或 **记录告警**（实现时二选一，建议在日志中标明）。

**天空键（Directional）**：方向键在 c2s 中仍有「附着/参考地面位置」的列；若无单独规则，**与地面相同**：用同一公式把 PJSK 参考列映射到 `2…13`。

## 2. CHUNITHM c2s 事件类型（示意图 + 样本归纳）

下列为 **地面 / 天空** 常见事件码（列名用示意图用语，实现时以真实 c2s 列数为准）：

| c2s 类型 | 示意图简述 | 备注 |
|----------|------------|------|
| `TAP` | 普通红键 | 地面单点 |
| `FLK` | 普通蓝键 / Flick | 常带方向后缀 `L`/`R` 等 |
| `CHR` | 普通黄键 / 侧键类 | 可带 `UP` 等 |
| `HLD` | 长押 | 比 TAP 多持续时间（tick） |
| `SLC` / `SLD` | 滑条段 | 多段链式，与 `SLD` 串接 |
| `SXC` / `SXD` | 滑条变体 | 示意图列出；与 SLC/SLD 分工 **待你定** |
| `AIR` | 向上天空 |  |
| `ADW` | 向下 |  |
| `ADR` / `ADL` / `AUL` / `AUR` | 斜向天空 | 与示意图命名一致 |
| `ALD` | 大天空长条 | 带持续时间 |
| `ASD` / `ASC` / `AHD` | 空中线 / 斜线 / 空长押 | 参数复杂，常含 `5.0`、`DEF` 等 |

## 3. 主映射表（SUS → c2s）**← 已定键型；轨位见 §1.5**

### 3.1 按 SUS「通道类型」映射（已定）

| SUS 通道（§1） | c2s 事件 | 说明 |
|----------------|----------|------|
| `1x` Tap | **`TAP`** | 所有 Tap 子类型均输出 `TAP`（见 §3.2） |
| `2xy` Hold | **`HLD`** | 同通道拼条，算持续 tick；中继点是否拆段见 §3.4 |
| `3xy` Slide 1 | **`SLC`** | 一条滑条可对应多行 `SLC`；官方 c2s 常配合 **`SLD`** 串段，实现时应对齐 `2896_04.c2s` 链式写法（见 §3.5） |
| `4xy` Slide 2 | **`SLC`** | 与 Slide 1 **相同策略**（均走 SLC 系，不用 `SXC`/`SXD` 除非日后另定） |
| `5x` Directional | **`AIR` / `ADW` / `ADR` / `ADL` / `AUL` / `AUR`** | 与 §3.3 一一对应 |

### 3.2 Tap：`#mmm1x` 数据首字符 → c2s（已定）

| SUS Tap 子类型（首字符） | → c2s |
|---------------------------|-------|
| `1` | `TAP` |
| `2` | `TAP` |
| `3` | `TAP` |
| `4` | `TAP` |
| `5` | `TAP` |
| `6` | `TAP` |

> 不区分 PJSK 子类型颜色：**一律 `TAP`**。

### 3.3 Directional：`#mmm5x` 数据首字符 → c2s（已定）

| SUS 方向（首字符） | 规格含义 | → c2s |
|--------------------|----------|-------|
| `1` | 上 | `AIR` |
| `2` | 下 | `ADW` |
| `3` | 左上 | `AUL` |
| `4` | 右上 | `AUR` |
| `5` | 左下 | `ADL` |
| `6` | 右下 | `ADR` |

### 3.4 Hold：`#mmm2xy`（实现提示）

| SUS 点类型 | 处理 |
|------------|------|
| `1` 起点 | 与同通道 `2` 终点合并输出 **一行 `HLD`**（持续 tick = 终时刻 − 起时刻） |
| `2` 终点 | 同上 |
| `3` 中继 | **首版**可简化为折线近似：仍输出单行 `HLD`（取首末时刻与首末位置），或按中点拆多段 `HLD`（精度更高，实现更复杂） |

### 3.5 Slide：`#mmm3xy` / `#mmm4xy`（实现提示）

- **语义**：SUS 侧 Slide 1 / Slide 2 **均**转为中二 **`SLC`**（及格式所需的 **`SLD`** 链）。  
- **几何**：将贝塞尔/控制点折线为多段折线，每段或每节点对应一行 `SLC`，段间关系按官方样本用 `SLD` 衔接。  
- 控制点 `4`、不可见中继 `5`：参与插值，**不一定**单独占一行可见 `SLC`。

## 5. c2s 侧「无直接 SUS 来源」的类型

下列多出现在 **CHUNITHM 侧组合技** 或 **编辑器生成**，**SUS 可能没有一一对应源**；默认 **不由纯 SUS 自动生成**，除非后续规则补充：

| c2s 类型 | 说明 |
|----------|------|
| `ASC` / `ASD` / `AHD` | 常与 `HLD`、其它天空线联动；示意图含 `5.0`、`DEF` |
| `ALD` | 大天空条；是否由某类 SUS 长条+通道推导：**待定** |
| `FLK` 方向后缀、`CHR` 的 `UP` 等 | 依赖 Tap 子类型映射或额外启发式 |
