# Event 类型解读（基于现有 ACUS 样本）

本文档基于 `desktop/ACUS/event/**/Event.xml` 的 `substances/type` 实际样本做“经验性解读”。
不同版本可能存在新增类型或语义变化，请以目标版本资源为准。

## 已实现的筛选分类

- `ULT/WE曲解锁`：`type=3` 且事件名包含 `ULT` 或 `WE`（已在管理页合并为一个分类）
- `地图解禁`：`substances/map/mapName.id != -1`
- `宣传(含DDS)`：`substances/information/image/path` 指向存在的 DDS 文件
- `其它`：不满足以上规则

## type 对照（样本推测）

- `1`：信息告知类（曲告知/地图告知）
- `2`：地图开关/地图标志（样本名含“マップフラグ”）
- `3`：乐曲相关开锁/常驻（含 ULT/WE、曲开放、曲常駐）
- `4`：广告影片（アドバムービー）
- `5`：推荐乐曲（おすすめ曲）
- `6`：在线标志/在线开关（オンラインフラグ）
- `7`：课程开锁（コース開放）
- `8`：任务/Quest 标志（クエストフラグ）
- `10`：头像相关更新（アバター更新）
- `11`：权利表记（権利表記）
- `12`：游玩报酬开关（プレイ報酬フラグ）
- `13`：日替奖励预设（日替わりボーナス）
- `14`：任务/挑战相关标志（样本含ミッションフラグ）
- `17`：Linked GATE / LinkedVerse 相关

## 判定建议

- 要做“开图”活动，与 A001 Bandori（Ave Mujica / MyGO）一致时：用 **`type=2`（マップフラグ）** + `substances/map/mapName`；`information/mapFilterID` 为 Invalid。不要用 `type=6` 冒充开图（本仓库旧版生成器曾误用）。
- 要做“曲相关”活动，优先关注 `type=3`，并根据名称再细分 ULT/WE/常驻
- 管理页分类属于“运营便利分类”，不是官方 schema 的强约束
