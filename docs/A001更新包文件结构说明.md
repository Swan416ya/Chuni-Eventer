# CHUNITHM A001 更新包文件结构说明

本文档描述音游 CHUNITHM 一次更新包（A001 文件夹）内的文件结构与关键数据格式，便于后续开发「自定义 Map + 自定义 Character + 自定义 CourseMusic」生成器时参考。

---

## 一、顶层结构概览

```
A001/
├── data.conf                 # 版本与数据标识（CHUNITHM 2.45.1）
├── avatarAccessory/          # 头像挂件

├── chara/                    # 角色（Character）

├── cmission/                 # 任务

├── ddsImage/                 # DDS 图片资源（角色立绘等）

├── emoteScene/               # 表情场景

```

与「自定义 Map + 自定义 Character + 自定义 CourseMusic」强相关的是：**map**、**mapArea**、**mapBonus**、**course**、**chara**、**music**、**reward**、**releaseTag**、**ddsMap**、**cueFile**、**event**（地图显示/解锁由活动控制）等。

## 二、根配置文件：data.conf

- **路径**：`A001/data.conf`
- **作用**：标识该包所属游戏与版本。
- **示例**：
  - `Name = CHUNITHM`
  - `VerMajor = 2`, `VerMinor = 45`, `VerRelease = 1`

自定义包可沿用或修改版本号以区分。

---

## 三、Map（地图）

- **目录**：`A001/map/`
- **结构**：每个地图一个子目录，目录名 = `map` + 8 位数字 ID（如 `map03020814`），内含 `Map.xml`。
- **根元素**：`<MapData>`。

### 3.1 主要字段


| 字段                                                        | 说明                                                            |
| --------------------------------------------------------- | ------------------------------------------------------------- |
| `dataName`                                                | 与目录名一致，如 `map03020814`                                        |
| `name`                                                    | 显示名（id + str），如 id=3020814, str="Mystic Rainbow of X-VERSE-X" |
| `mapType`                                                 | 地图类型（0/1 等）                                                   |
| `hiddenType`                                              | 隐藏类型                                                          |
| `unlockText`                                              | 解锁说明文案                                                        |
| `mapFilterID`                                             | 地图筛选 ID（如 Current/Other）                                      |
| `categoryName` / `timeTableName` / `stopReleaseEventName` | 分类、时间表、停止释放事件（可为 Invalid）                                     |
| `priority`                                                | 排序用优先级                                                        |
| `infos`                                                   | **核心**：`MapDataAreaInfo` 数组，描述地图上每个「区域」                       |


### 3.2 MapDataAreaInfo（地图区域信息）

每个区域对应地图上的一个节点/格子，包含：


| 字段                          | 说明                                                 |
| --------------------------- | -------------------------------------------------- |
| `mapAreaName`               | 引用 **mapArea** 的 ID/str（如 3232001 / "VERSE通常2001"） |
| `ddsMapName`                | 引用 **ddsMap** 的贴图（如 0="共通0001_CHUNITHM" 或 4907）    |
| `musicName`                 | 该格 BGM，多为 -1（Invalid）                              |
| `rewardName`                | 该格 **reward** 的 ID/str（如 20008000）                 |
| `isHard`                    | 是否困难路线                                             |
| `pageIndex` / `indexInPage` | 分页与页内索引                                            |
| `requiredAchievementCount`  | 达成数要求                                              |
| `gaugeName`                 | 血量/ gauge 规则 ID                                    |


**要点**：Map 不直接包含「课题曲目列表」，只通过 `mapAreaName` 关联到 **mapArea**，通过 `rewardName` 关联到 **reward**。若要和「课题曲目」挂钩，需要靠 **mapArea** / **mapBonus** 或活动设计里再引用 **course**。

### 3.4 与 Event 的关系（显示/解锁）

**地图在游戏内的显示与解锁由 Event（活动）控制。** 仅配置 Map + MapSort 不足以让地图出现在游戏里，需要有一个 **Event** 引用该地图或对应 mapFilterID，玩家在活动入口进入后才会看到并解锁该地图。

- Map 侧的 `mapFilterID`、`stopReleaseEventName` 等可与活动/时间表配合。
- Event 侧通过 `substances.map.mapName` 指定「本活动使用的地图」（map 的 name.id）；部分活动类型下还会用 `substances.information.mapFilterID` 做地图筛选。
- 因此做**自定义 Map** 时，除 Map、MapSort、MapArea、Reward 等外，还需至少配置一个 **Event**，在其 `substances.map.mapName` 中填入该 Map 的 `name.id`，并在 `event/EventSort.xml` 中登记该 Event，地图才会在活动中显示并可玩。

### 3.5 MapSort.xml

- **路径**：`A001/map/MapSort.xml`
- **根元素**：`<SerializeSortData>`，`dataName="map"`。
- **内容**：`SortList` 下多个 `StringID`，每个 `id` 为地图的 name.id，用于 UI 排序。生成新 Map 时若要在列表中显示，需在此增加对应 id。

## 四、MapArea（地图区域）

- **目录**：`A001/mapArea/`
- **结构**：`mapArea` + 8 位数字 ID（如 `mapArea03232001`），内为 `MapArea.xml`。
- **根元素**：`<MapAreaData>`。

### 4.1 主要字段


| 字段                                          | 说明                                  |
| ------------------------------------------- | ----------------------------------- |
| `dataName`                                  | 如 `mapArea03232001`                 |
| `name`                                      | id 与 dataName 中的数字一致（如 3232001）     |
| `mapBonusName`                              | 引用 **mapBonus**（该区域的加成规则）           |
| `mapAreaBoostType` / `mapAreaBoostMultiple` | 加成类型与倍数                             |
| `grids`                                     | **MapAreaGridData** 数组：格子类型、出入口、奖励等 |


Map 的 `MapDataAreaInfo.mapAreaName.id` 与 MapArea 的 `name.id` 对应（如 3232001 → mapArea03232001）。

---

## 五、MapBonus（地图加成）

- **目录**：`A001/mapBonus/`
- **结构**：`mapBonus` + 8 位数字（如 `mapBonus00920172`），内为 `MapBonus.xml`。
- **根元素**：`<MapBonusData>`。

### 5.1 与自定义 Character / Music 的关系

`MapBonusData.substances` 中有 **MapBonusSubstanceData**，其中可配置：

- `chara`：指定角色（charaName）及点数
- `music`：指定曲目（musicName）及点数
- 以及 `charaWorks`、`skill`、`musicGenre`、`musicWorks`、`musicLabel`、`musicDif`、`musicLv`、`charaRank` 等

因此：**自定义角色、自定义曲目**可以通过 Map → MapArea → MapBonus 这条链，在「地图加成」里被引用。

## 六、Course（课题）与 CourseMusic

- **目录**：`A001/course/`
- **结构**：每个课题一个子目录，如 `course00500601`、`course00045001`，内为 `Course.xml`。
- **根元素**：`<CourseData>`。

### 6.1 主要字段


| 字段                                     | 说明                                                |
| -------------------------------------- | ------------------------------------------------- |
| `dataName`                             | 如 `course00500601`                                |
| `releaseTagName` / `netOpenName`       | 版本与网络开放标签                                         |
| `name`                                 | 课题名（id 如 500601，str 如 "Linked Gate NEW-1"）        |
| `difficulty`                           | 难度（如 CLASS Ⅰ～Ⅴ）                                   |
| `rule`                                 | 规则 ID                                             |
| `reward` / `reward2nd`                 | 通关奖励（reward 的 id）                                 |
| `teamOnly` / `isMusicDuplicateAllowed` | 是否仅队伍、是否允许曲目重复                                    |
| `conditionsCourse` / `conditionsText`  | 前置课题、条件文案                                         |
| `priority`                             | 排序                                                |
| `**infos`**                            | **CourseMusicDataInfo 数组 = 课题内曲目列表（CourseMusic）** |


### 6.2 CourseMusicDataInfo（课题内单曲）


| 字段                | 说明                                                                   |
| ----------------- | -------------------------------------------------------------------- |
| `type`            | 类型（0 等）                                                              |
| `selectMusic`     | **musicName**（id/str）+ **musicDiff**（Basic/Advanced/Expert/Master 等） |
| `selectLevel`     | 等级筛选（常为 Invalid）                                                     |
| `selectMusicList` | 可选曲列表（可空）、panelType、isRecordShown                                    |


**要点**：  

- 自定义 **CourseMusic** = 在 Course 的 `infos` 里增加/修改 `CourseMusicDataInfo`。  
- 每一条的 `selectMusic.musicName.id` 指向 **music** 的 name.id；`musicDiff.id` 一般为 0～3（Basic/Advanced/Expert/Master）。

### 6.3 CourseSort.xml / CourseDifSort.xml

- `CourseSort.xml`：课题的排序列表（StringID 的 id 为 name.id）。
- `CourseDifSort.xml`：按难度等排序用。  
生成新 Course 时若要在游戏内列表显示，需在对应 Sort 中登记 id。

## 七、Chara（角色）

- **目录**：`A001/chara/`
- **结构**：`chara` + 6 位数字（如 `chara024690`），内为 `Chara.xml`。
- **根元素**：`<CharaData>`。

### 7.1 主要字段


| 字段                                          | 说明                         |
| ------------------------------------------- | -------------------------- |
| `dataName`                                  | 如 `chara024690`            |
| `name`                                      | 角色名（id 24690, str "結束いのり"） |
| `sortName`                                  | 排序用名称                      |
| `works`                                     | 所属作品（charaWorks）           |
| `defaultHave` / `rareType` / `ranking`      | 默认拥有、稀有度、是否排行              |
| `defaultImages` / `addImages1`～`addImages9` | 默认立绘与追加立绘（引用 ddsImage 等）   |
| `ranks`                                     | CharaRankData：等级与奖励技能种子等   |


**引用关系**：  

- MapBonus 的 substances 里可指定 `charaName`（id）引用 Chara。  
- Reward 里也可发放角色（charaName）。  
自定义角色需在 **chara**、**ddsImage**（立绘）等处有一致 ID 与命名。

---

## 八、Music（曲目）

- **目录**：`A001/music/`
- **结构**：`music` + 数字 ID（如 `music2900`），目录内通常包含：
  - `Music.xml`
  - 谱面文件：`{id}_00.c2s`～`{id}_03.c2s`（Basic/Advanced/Expert/Master）
  - 封面：如 `CHU_UI_Jacket_2900.dds`
  - 音频相关：`music2900.acb`、`music2900.awb`（由 **cueFile** 引用）

### 8.1 Music.xml 根元素：`<MusicData>`


| 字段                                                      | 说明                                                         |
| ------------------------------------------------------- | ---------------------------------------------------------- |
| `dataName`                                              | 如 `music2900`                                              |
| `name`                                                  | 曲名（id 2900, str "一旦ステイ TONIGHT"）                           |
| `artistName` / `genreNames` / `worksName` / `labelName` | 艺术家、流派、作品、厂牌                                               |
| `jaketFile`                                             | 封面 DDS 路径                                                  |
| `cueFileName`                                           | 对应 **cueFile** 的 name（id/str）                              |
| `fumens`                                                | **MusicFumenData** 数组：各难度谱面 type、file、level、levelDecimal 等 |


### 8.2 与 Course、MapBonus 的关系

- **Course**：`CourseMusicDataInfo.selectMusic.musicName.id` = Music 的 `name.id`。
- **MapBonus**：substances 里可指定 `musicName`（id）对应 Music。  
自定义曲目需在 **music**、**cueFile**（及 acb/awb）、谱面 c2s、封面 dds 等保持 ID 与路径一致。

## 九、Reward（奖励）

- **目录**：`A001/reward/`
- **结构**：`reward` + 8 位数字（如 `reward070009718`），内为 `Reward.xml`。
- **根元素**：`<RewardData>`。

### 9.1 与 Map / Chara 的关系

- **Map**：`MapDataAreaInfo.rewardName.id` 指向 Reward 的 `name.id`。
- **Reward** 的 `substances` 里可包含：`chara`（charaName）、`trophy`、`skillSeed`、`namePlate`、`gamePoint`、`ticket` 等。  
自定义地图节点奖励 = 新建/引用 Reward，其中可发自定义角色等。

## 十、其他相关资源

### 10.1 releaseTag

- **路径**：`A001/releaseTag/releaseTag000020/ReleaseTag.xml`
- **作用**：版本标签（如 "v2 2.45.00"），Course / Chara / Music 等常引用 `releaseTagName`（id 20）。  
自定义包可沿用同一 releaseTag 或新建。

### 10.2 ddsMap

- **路径**：`A001/ddsMap/ddsMap00000000/DdsMap.xml`
- **作用**：地图贴图，`ddsFile.path` 指向 DDS 文件。Map 的 `MapDataAreaInfo.ddsMapName` 引用此处。

### 10.3 cueFile

- **路径**：`A001/cueFile/cueFile002900/CueFile.xml`
- **作用**：曲目音频，`acbFile` / `awbFile` 指向 acb/awb。Music 的 `cueFileName` 与这里对应（如 id 2900）。

### 10.4 ddsImage

- **路径**：`A001/ddsImage/ddsImage024690/` 等
- **作用**：角色立绘等 DDS，Chara 的 `defaultImages` / `addImages` 引用。

## 十一、Event（活动）与 Map 显示/解锁

- **目录**：`A001/event/`
- **结构**：每个活动一个子目录，如 `event00002512`、`event00018093`，内为 `Event.xml`。
- **根元素**：`<EventData>`。

### 11.1 主要字段


| 字段                                           | 说明                                                           |
| -------------------------------------------- | ------------------------------------------------------------ |
| `dataName`                                   | 如 `event00018093`                                            |
| `name`                                       | 活动名（id 如 18093，str 为活动标题）                                    |
| `periodDispType` / `alwaysOpen` / `teamOnly` | 期间表示、是否常开、是否仅队伍                                              |
| `substances`                                 | **核心**：活动内容块，`type` 表示活动类型（如 6=flag, 8=quest, 13=日替わりボーナス 等） |


### 11.2 substances 中与 Map 相关的部分

- `**substances.map.mapName`**：指定本活动使用的**地图**，填 Map 的 `name.id`（如 3020814）。若为 -1 表示本活动不挂载地图。
- `**substances.information.mapFilterID`**：与地图筛选相关，可与 Map 的 `mapFilterID` 配合，用于在活动内筛选显示哪些地图。

因此：**Map 要在游戏中显示并可供玩家解锁，必须被至少一个 Event 通过 `substances.map.mapName` 引用；** 自定义 Map 生成器需要同时生成或关联一个 Event，并登记到 `event/EventSort.xml`。

### 11.3 EventSort.xml

- **路径**：`A001/event/EventSort.xml`
- **作用**：与 MapSort/CourseSort 类似，`SortList` 下为各活动的 `name.id`，用于活动列表排序与显示。

## 十三、与「自定义 Map + Character + CourseMusic」的对应关系

1. **自定义 Map**
  - 在 `map/` 下新增 `mapXXXXXXXX/Map.xml`。  
  - 每个区域的 `mapAreaName` 指向已有或新建 **mapArea**；`rewardName` 指向已有或新建 **reward**；`ddsMapName` 指向 **ddsMap**。  
  - 在 `map/MapSort.xml` 中追加该 Map 的 `name.id`。  
  - **必须**在 `event/` 下新建或修改一个 **Event**，在其 `substances.map.mapName` 中填入该 Map 的 `name.id`，并在 `event/EventSort.xml` 中登记该 Event，地图才会在游戏内显示并可解锁。
2. **自定义 Character**
  - 在 `chara/` 下新增 `charaXXXXXX/Chara.xml`，并在 `ddsImage/` 准备立绘。  
  - 在 **MapBonus** 的 substances 中引用该 charaName；或在 **Reward** 中作为奖励发放，再被 Map 的 rewardName 引用。
3. **自定义 Music 作为 CourseMusic**
  - 在 `music/` 下新增 `musicXXXX/`（Music.xml + 谱面 c2s + 封面 dds），并在 **cueFile** 与音频资源中登记。  
  - 在 **course/** 下新建或修改 Course，在 `infos` 中增加 `CourseMusicDataInfo`，`selectMusic.musicName.id` 指向该 Music 的 id，并设置 `musicDiff`。  
  - 在 `course/CourseSort.xml`（及必要时 CourseDifSort.xml）中登记该 Course 的 id。
4. **地图与课题的衔接**
  - Map 本身不直接引用 Course；若要让「某地图」与「某课题」在玩法上关联，需通过活动设计（**event**）、任务（quest/cmission）或 MapBonus 中的 music/chara 等间接绑定。
5. **地图显示与 Event**
  - 地图在游戏中的显示与解锁由 **Event** 控制；仅做 Map + MapSort 不会让地图出现在游戏里，必须有一个 Event 通过 `substances.map.mapName` 引用该 Map。

以上即为 A001 更新包中与自定义 Map、Character、CourseMusic 相关的文件结构说明，可作为生成器设计与实现的参考。
